# 用户行为异常检测

## 背景

这是一个股票交易平台的用户行为日志样本。平台上用户会产生多种行为：查看行情、点击内容、关注股票、发起交易。

我们要从这份日志中找出**行为模式异常的用户**——他们可能是：
- 羊毛党（利用活动规则刷量的账户）
- 机器人（自动化脚本）
- 有价值的高阶用户（高频交易者、专业投资者）

异常不一定是"坏"——有些异常用户恰恰是高价值客户。你需要区分。

## 数据

文件：[`data.csv`](./data.csv)

- 10000 行
- 字段：
  - `user_id` — 用户 ID
  - `timestamp` — 行为发生时间
  - `action_type` — 行为类型，枚举值 `view` / `click` / `follow` / `trade`
  - `stock_code` — 股票代码
- 时间跨度：一个交易日的上午 09:30-11:30 和下午 13:00-15:00

## 任务

在限定时间内（通常 15-20 分钟），找出你认为异常的用户，说明：

1. 你怎么定义"异常"（可以有多种定义）
2. 找到了哪些用户（至少给出 user_id 列表）
3. 这些用户可能是什么画像
4. 你的置信度有多高；如果不确定，说出你需要什么额外信息才能确认

## 工具

不限。`pandas` / `sklearn` / SQL / Excel / 任何 AI 工具（Claude、Cursor、ChatGPT 等）都可以。我们不考察你对某个工具的熟练度，而考察你能多快用合适的工具把想法落地。

## 评分导向

- **思路 > 代码行数。** 能清晰讲清楚"为什么这样定义异常""为什么这个用户落入这类"，比跑出一堆数字更重要。
- **诚实边界。** 遇到拿不准的数据或结论，承认边界并说出还需要什么信息，比硬下结论更有价值。
- **业务推理 > 纯算法。** 异常检测的难点不在跑聚类，而在解读——"高 trade + 聚焦股票" 到底是羊毛党回测机器人，还是高净值专业投资者？这个判断需要你提出来。
