# 用户行为分类与处置建议

## 背景

这是一个股票交易平台的用户行为日志样本。平台上用户会产生多种行为：查看行情、点击内容、关注股票、发起交易。

平台运营团队需要从这份日志中识别**值得关注的用户**，并建议如何处置——有些需要风控限制或封号，有些反而是应该主动留住的高价值客户。你的任务是做这个判断。

## 数据

文件：[`data.csv`](./data.csv)

- 10000 行
- 字段：
  - `user_id` — 用户 ID
  - `timestamp` — 行为发生时间
  - `action_type` — 行为类型，枚举值 `view` / `click` / `follow` / `trade`
  - `stock_code` — 股票代码
- 时间跨度：一个交易日的上午 09:30-11:30 和下午 13:00-15:00

## 任务

把 506 个用户分到 5 个桶：

- **A. 羊毛党要封号** —— 明显违规或滥用的账户
- **B. 机器人要风控限流** —— 自动化脚本，非正常人类操作
- **C. 高价值客户要 VIP 服务** —— 高频交易者、专业投资者等应主动留住的用户
- **D. 正常用户** —— 无需特殊处理
- **E. 不确定** —— 现有数据不足以定类

### 交付内容

1. 你的桶划分方法——说清楚定义每个桶的判据
2. 每个桶的用户 ID 列表（A / B / C / E 桶必须列全；D 桶数量大，不要求全部列出）
3. **从 E 桶挑 1 个用户，详细说明：** 他看起来像什么？你需要什么额外数据才能把他从 E 移到 A / B / C / D 中的某一桶？

### 时间

限定时间内完成（通常 15-20 分钟）。不追求完美，追求推理清晰。

## 工具

不限。`pandas` / `sklearn` / SQL / Excel / 任何 AI 工具（Claude、Cursor、ChatGPT 等）都可以使用。

## 评分导向

- **业务判断 > 模式识别。** "把这些看起来异常的用户都扔进 A 桶" 是简单的异常检测，不是这个任务要的答案。A / B / C 的区分需要你理解业务代价——封错一个 VIP 客户的代价远大于漏掉一个羊毛党。
- **承认不确定 > 强行分类。** E 桶不是逃避，是承认边界。把模糊用户扔进 E 桶并说清楚你需要什么信息才能定类，比硬分到 A 或 C 更有价值。
- **判据清晰 > 结论多少。** 分类的数量不重要，每个桶的判据能讲清楚才重要——要能解释"为什么这个用户分到这里"以及"为什么不分到其他桶"。
